\section{Why Pixel-Space Action Helps Preserve Video Pretraining}

\todo{建议篇幅：600--800 words，4--5 段。这是 Discussion 的核心新小节。
\begin{enumerate}
  \item P1: 从 VLM 类比开始。Pretraining 能力更容易迁移到保留原生 output interface 的 post-training 任务。
  \item P2: 分析 video model。Web video pretraining 学到 scene dynamics, object motion, temporal continuity；如果 post-training 输出变成 action head/vector，这些能力需要经过额外接口才能影响 action。
  \item P3: 解释 Optical Action Encoding。WAN 仍然生成 video latents/frames，action supervision 通过 action-labelled pixels 进入同一个生成目标。
  \item P4: 讨论这带来的好处：保留 video-generation prior、让 action label 可视化、降低 pure IDM 从 frame difference 反推动作的歧义。
  \item P5: 控制边界：action decoder 仍然是额外模块，所以本文不是“无 decoder”，而是“让 video model 显式生成可读 action signal”。
\end{enumerate}}

\section{Action Decoder Robustness and Generated-Frame Gap}

\todo{建议篇幅：500--750 words，4 段。
\begin{enumerate}
  \item P1: 解释 decoder 是关键瓶颈。Training frames, noisy-label frames, WAN generated frames and simulation frames 的分布不完全一致。
  \item P2: 讨论 noisy trajectory layer。它的目的不是制造更好看的图，而是让 decoder 在 visual label 有偏时仍结合 RGB and geometry cues。
  \item P3: 讨论 generated-frame gap。WAN 生成帧可能在视觉上合理，但 keypoint depth、trajectory curvature or object contact relationship 可能偏离可执行动作。
  \item P4: 讨论 rollout finetuning or generated-frame decoder finetuning 的风险。Open-loop error 可能改善，但 closed-loop 不一定改善，因为 closed-loop context 会随 action 改变。
\end{enumerate}}

\section{Open-Loop and Closed-Loop Gap}

\todo{建议篇幅：450--650 words，4 段。
\begin{enumerate}
  \item P1: 解释 open-loop 为什么必要。它可以隔离 generation/decoding errors，并在不运行 simulation 的情况下快速比较 checkpoints。
  \item P2: 解释 open-loop 为什么不充分。Closed-loop 中每个 action 会改变 future observation，error accumulation and exposure bias 会放大微小误差。
  \item P3: 分析具体来源。Domain mismatch, CFG guidance, temporal alignment, camera calibration, object contact dynamics and decoder error 都可能造成 open-loop/closed-loop mismatch。
  \item P4: 说明本文如何应对。Block protocol, policy server, guidance sweep, qualitative failure inspection, and per-task success logging 让 mismatch 可诊断。
\end{enumerate}}

\section{Limitations}

\todo{建议篇幅：450--650 words，5 段或分点。态度要诚实。
\begin{enumerate}
  \item P1: Dataset limitation。主结果主要基于一个具体 robot manipulation experimental setting，更多平台和数据源仍是 future work。
  \item P2: Evaluation limitation。Closed-loop tasks and runs 可能有限，real robot evaluation 不一定完成。
  \item P3: Baseline limitation。完整 VLA/action-head、WAM/IDM、same-backbone readout 对照需要更多训练资源，FYP 中可能只能做 partial or conceptual comparison。
  \item P4: Technical limitation。Codec 依赖 camera calibration；generated visual labels 可能有 depth/trajectory drift；decoder error 会影响 control。
  \item P5: Compute limitation。WAN training/eval 依赖 GPU and ACP resources，复现实验成本较高。
\end{enumerate}}
